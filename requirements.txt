# ============================================================
# COLAB COMPATIBILITY NOTE (v2.9.4)
# ============================================================
# Colab's runtime ships with numpy 2.x and a stack of packages
# (datasets, sklearn, scipy, transformers) compiled against it.
# Previous M.I.N.E.R.V.A. versions pinned numpy<2.0 out of caution
# inherited from the 2024 numpy 2.0 release. That cap is now
# actively HARMFUL on Colab because:
#   1. pip downgrades numpy to 1.x
#   2. preinstalled packages still hold the 2.x ABI in memory
#   3. import raises ValueError: dtype size changed
#   4. half the pipeline fails to import
#
# As of 2026, the entire dependency stack (feyn 3.5.0, datasets 2.21+,
# transformers 4.46+, sklearn 1.5+, torch 2.5+) supports numpy 2.x.
# So we now ALLOW numpy 2.x and let the rest of the stack ride on
# whatever Colab provides. This eliminates the kernel-restart dance.
#
# Python compatibility:
#   Production:  Python 3.11 (Colab default) — fully tested
#   Supported:   Python 3.10, 3.12
#   3.13:        feyn has no 3.13 wheels; pipeline auto-falls back
#                to sklearn LogReg in scripts/08 (architecturally
#                equivalent for downstream scoring).
# ============================================================
