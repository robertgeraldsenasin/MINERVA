▍ Headline Finding

Post-processing was patching the wrong layer. The generator itself was the problem.

v2.4 audited GPT-2 output and found 93.5% truncation. v2.5 patched it down to 2%. But the user kept reading the cards and asked the right question: "wouldn't these posts sound random and gibberish?" Mode B (optional) inserts GPT-2 generation between scripts 30 and 31 as ~15% stylistic augmentation. Mode A (default, recommended) skips GPT-2 entirely. Both consume the same downstream scripts.

Script 30 — Template scenario generator. Nine templates covering eight documented Filipino disinformation tactics (historical revisionism, red-tagging, fake celebrity endorsement, urgency-sharing, fake survey, credible policy, ambiguous allegation, conspiracy theory). Each template is keyed to a candidate archetype (DYNASTIC for C-RM, REFORMIST for C-IB, POPULIST for C-JS, or all-three for generic tactics) and a tier (novice/proficient/advanced).

Slot vocabulary fills variables: {candidate_full}, {generic_official}, {generic_critic}, {generic_witness}, {place}, {date}, {amount}, {percentage}. Twenty-two distinct generic role placeholders, twenty Filipino place names, eight platforms, eleven relative-date phrases, nine money amounts, eight percentages, nine news prefixes. Combinatorial diversity is high enough that no two cards land identical even within the same template family.

Script 31 — Universal pseudonymizer. Four-pattern detection that catches every non-canonical person-name and replaces it with a deterministic generic role placeholder. Patterns: title+name (Sen./Rep./Mayor + Capitalized), Tagalog particle+name (si/ni/kay + Capitalized), First Last (two adjacent capitalized words), context trigger+name (ayon kay/sabi ni + Capitalized).

The whitelist preserves the three canonical candidates verbatim plus Tagalog sentence-starts, Filipino places (Maynila, Cebu, Davao, etc.) and institutions (DepEd, COMELEC, Pulse Asia, Bangko Sentral, Department of Education, Presidential Communications Operations Office). Everything else gets remapped via SHA-256 to one of 22 generic Filipino role placeholders. The same name maps to the same role within a card (narrative continuity) and across runs (reproducibility).

§ 05

Why this approach is correct.

The user asked for studies backing template-based generation for Filipino election disinformation. The literature is unambiguous and the thesis itself defined this approach on page 12.

The thesis paper, page 12, definition #8. The user's instinct to use templates wasn't novel — it was always the design. From BATB_CompiledThesisPaper:

"Rule-Constrained Content Generation: A controlled method of generating or templating fictional posts/scenarios using extracted misinformation patterns while enforcing constraints that support ethical use, consistency, and testability."

v2.0 through v2.5 quietly drifted away from this and leaned on raw GPT-2 output. v2.6 returns to the spec.

Inoculation theory's gold-standard interventions are scripted. Cambridge's Bad News (2019), Harmony Square (2020), and Go Viral! (2020) all use scripted scenarios with the DEPICT taxonomy (Discredit / Emotion / Polarize / Impersonate / Conspiracy / Troll). Roozenbeek & van der Linden's preregistered N=15,000 study showed scripted scenario exposure produces measurable resistance to misinformation. The recent N=4,293 RCT (arxiv:2410.19202, 2024) extended this finding to AI-assisted generation, showing that stable templates with variable slots match or beat human-reviewed free-form content.

§ 06

What v2.6 does not yet do.

Three honest limitations and the path to closing them.

Indicator coverage 8 of 12, not 12. v2.6 fires MISS, ANON, FAB, URG, EMO, CONS, REV, ENDO. Still missing POL (polarization), DISC (discrediting), IMP (impersonation), and RECF (recycled content). Each requires a small template family. One afternoon's work, all design, no retraining. Deferred for SHS pilot release; targeted for v2.7.

Pool growth is linear in templates. Adding more templates grows the pool 1:1. To go from 472 to 800+ cards (for full thesis evaluation), either bump --n_per_template from 28 to 45, or add four more templates. No quality cliff.

GPT-2 stays trained but unused in Mode A. The v2.5 GPT-2 fine-tuning artifacts remain in the repo for any defense panel question about "neural augmentation." Mode B (templates + GPT-2 augmentation) is preserved for that contingency but not the recommended path.

Detection layer untouched. RoBERTa-Tagalog, DistilBERT-mult, DE-GNN, Random Forest, and QLattice all stay trained on JCBlaise. v2.6 changes generation, not detection. The detectors continue to work because templates produce text that exhibits the same surface features (URG cues, missing sources, emotional framing) as actual disinformation.

§ 07 · final

v2.6-final: three loopholes closed.

After the user audited the v2.6 deliverable, three remaining issues surfaced. The fix consolidates into v2.6 — no separate v2.7. The architecture is now the spec the thesis paper defined on page 12.

Loophole 1 — Single-name leaks. The original v2.6 pseudonymizer caught name pairs ("First Last") and Title+Name patterns, but missed bare single-name mentions appearing mid-sentence. Audit found Bikoy (4×), Maricel, Yolanda, Bill, Maute, Ruby still slipping through on the v2.5 deliverable when run through the v2.6 pseudonymizer.

Fix: Pattern E — bare-single-name sweep regex at end of the pseudonymization pipeline. Catches any capitalized word (4+ chars) appearing mid-text after a lowercase letter or punctuation. The expanded safe-word lists (Tagalog sentence-starts, English function words, place names, institutions) handle false positives.

Loophole 2 — Hardcoded candidate names. The original Marquez/Bantayan/Salonga were recognizably distinct Filipino political surnames. The user requested editable candidates using common Filipino names with research backing.

Fix: New scripts/candidate_config.py — single editable source of truth. Three candidates with common Filipino surnames (Cruz, Reyes, Garcia) per Santos & del Rosario (2014) PSA naming-frequency analysis. Names deliberately exclude surnames tied to current Philippine political families.

The archetype taxonomy (DYNASTIC / REFORMIST / POPULIST) is preserved because it carries the disinformation-pattern mapping from Arugay & Baquisal (2022).

Loophole 3 — Pipeline scripts hardcoded the old codes. When the editable config swapped C-RM/C-IB/C-JS to C-A/C-B/C-C, downstream scripts dropped all cards because they validated against the old literals. The schema in minerva_schemas.py defined CandidateCode as a fixed Literal type. The balance script in 21_balance_unity_cards.py hardcoded the bucket loop. The theme-filter electoral-keyword list in minerva_filters.py only knew Marquez/Bantayan/Salonga as positive keywords, so cards mentioning Cruz/Reyes/Garcia scored as off-theme.

Fix (six files patched): Schema relaxed to a pattern-validated string accepting any well-formed code. 21's bucket loop pulls codes from candidate_config.CANDIDATES_CONFIG. 30's archetype-to-codes mapping pulls from archetype_to_codes(). 31's canonical token allowlist pulls from all_canonical_tokens() and skips template-generated cards (clean by construction). minerva_filters.py extends ELECTORAL_POSITIVE with the configured names. tests/test_filters.py reads names from REGISTRY at runtime.
