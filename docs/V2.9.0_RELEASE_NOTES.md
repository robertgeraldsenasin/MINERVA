# M.I.N.E.R.V.A. v2.9.0 — Release Notes

**Tag:** `v2.9.0`
**Branch:** `refine/v2.9.0-audit-fixes`
**Built:** 2026-05-08 in response to the v2.8.7 independent audit
**Type:** Refinement release. No new architecture; closes specific gaps the audit identified.

---

## Why this release exists

The v2.8.7 run produced its first non-zero GPT-2 contribution to the pool (64/642 cards = 10.0%) but the independent audit identified five concrete issues blocking a defensible thesis presentation:

1. The v2.8.6 percentile-binning patch never activated in the build (corpus report stamped `v2.6.final` with fixed thresholds 0.6/0.8 and 96-98% bin imbalance).
2. 21 real Filipino names leaked into the strict allowlist report, including 4 city names ("Lapu-Lapu City", "Baguio City", "Pasay City", "Metropolitan Manila") and the politician name "Duterte".
3. Faithfulness audit regressed from 100% (v2.8.5) to 87.18% (v2.8.7) — 231 indicator-phrase mismatches caused by GPT-2 cards inheriting a stub explanation block.
4. Explanation diversity at 8.5% (56 unique strings across 663 cards) — well below any reasonable pedagogical threshold.
5. The `reports/det.json` "100% accuracy on the pool" line is tautological; the system was evaluating detectors on the same cards they were used to score.

v2.9.0 addresses all five with code, not promises.

## What changed

### NEW: `scripts/35_pseudonymize_places.py`

Adds a second pseudonymization stage that runs after script 31's person-only pseudonymizer. Targets Philippine geographic entities the existing NER misses. Output: `generated/cards_pseudo_places.json`.

- Reads from `templates/places_blocklist.txt` (17 regions, 81 provinces, 80+ cities, common landmarks — all hand-curated).
- Deterministic mapping: same real entity gets the same pseudonym across all cards in a run.
- Pseudonyms parallel "Candidate A/B/C": cities → "City W/X/Y/Z", provinces → "Province L/M/N/O", regions → "Region I/II/III/IV", landmarks → "[Landmark]".
- Per-card audit metadata (`metadata.place_pseudonyms`) preserved for downstream auditing.
- Report: `reports/pseudo_places.json` with category breakdown + top-30 replaced names.

### NEW: `templates/places_blocklist.txt`

171 lines of hand-curated Philippine geographic entities. Coverage validated by 9 unit tests (`test_pseudonymize_places.py::TestCoverage`) including a regression test that asserts every name from the v2.8.7 audit's leak report is in the list.

### NEW: `templates/response_bank_v2.json` (replaces v1)

- 75 keys × 3 variants per key = **225 phrase entries**.
- Schema: `{indicator}/{role}/{tier}` where role ∈ {fake, real} and tier ∈ {novice, proficient, advanced}.
- Each entry has `phrase_tl` (Tagalog primary), `phrase_en` (English subtext), `verifier_action` (concrete verification step).
- 72% diversity ratio (162 unique TL phrases / 225 total) — way above the 30% audit recommendation.
- EMO and URG sections are entirely hand-authored; the remaining indicators use parametric templates that vary tone by tier (novice = direct, advanced = strategic). Hand-authoring the rest is recommended for Thesis 3 polish but is not a v2.9.0 blocker.

### UPDATED: `scripts/29_merge_gpt2_into_pool.py`

- `_build_explanation()` rewritten to consume the v2.9 response bank instead of emitting `"GPT-2 generation flagged X"` stubs. This directly addresses the 231-mismatch regression.
- `card_idx` parameter added so phrases rotate across cards (variant selection by `idx % len(variants)`). Two cards with identical fired indicators now get different phrases — pushes pool-level explanation diversity above 30%.
- NEW pre-merge filter: `gpt2_indicators_supported_by_bank()` checks every GPT-2 card's fired indicators against the response bank's keys. Cards with unsupported indicators (which would produce mismatched phrases) are dropped at the merge stage. Reject reason: `response_bank_coverage`.
- For real-verdict cards, automatically appends a CREDIBLE/real/{tier} phrase, mirroring the template generator's behavior.

### UPDATED: `scripts/10b_prepare_gpt2_neurosymbolic.py`

- Report now stamps `version: "v2.9.0"` (was `v2.6.final` — the visible signal that the v2.8.6 patch hadn't been applied).
- Report `thresholds` block now records the **actually-used** percentile thresholds (not just the configured fixed fallbacks). Auditors can confirm activation by reading the report.
- New audit assertion: if `bin_strategy=percentile` is on but the dominant bin is still >70% of training data, a loud warning is logged. This catches the failure mode where percentile binning runs against a degenerate input distribution.
- Report includes `audit_dominant_bin_pct` for direct audit access.

### NEW: `scripts/37_holdout_detector_eval.py`

Replaces the tautological `reports/det.json` "100% on pool" claim with a real generalization measurement.

- Reads a hand-labeled CSV (`templates/holdout_gpt2_labeled.csv`).
- Loads trained RoBERTa + DistilBERT (gracefully degrades to NaN if unavailable).
- Computes binary precision/recall/F1/accuracy per detector + per-card predictions.
- Output: `reports/holdout_detector_eval.json` — the panel-facing generalization number.
- Pure-Python metrics (no sklearn dependency) so it runs in lightweight environments.

### NEW: `templates/holdout_gpt2_labeled.csv` + `.README.md`

A 50-card seed holdout drawn from the user's actual v2.8.7 GPT-2 outputs (25 fake-target + 25 real-target). The `true_label` column is intentionally empty — the team hand-labels it. The README documents the protocol (single-annotator minimum, double-annotator recommended, Cohen's kappa for the rigorous version).

### UPDATED: `notebooks/MINERVA_Run_Colab_v2.9.0.ipynb`

- Version bumped throughout.
- New cell after universal pseudonymize: invokes script 35 (place-name pseudonymization).
- Balance cell updated to read `generated/cards_pseudo_places.json` (was `cards_pseudo.json`).
- New cell after strict allowlist: conditionally invokes script 37 if a holdout CSV exists.
- Updated commentary in cell 2 explaining each v2.9.0 change.

## How to verify v2.9.0 actually applied

After running the pipeline, three reports prove v2.9.0's fixes activated:

1. **`reports/gpt2_neurosymbolic_corpus.json`**: top-level `version` should read `"v2.9.0"` (NOT `"v2.6.final"`). The `audit_dominant_bin_pct` field should be ≤70%. If you see `v2.6.final` here, the 10b script wasn't updated.

2. **`reports/pseudo_places.json`**: should exist. `total_replacements` > 0 if any cards mentioned PH places. The first run on JCBlaise-derived cards should show ≥50 replacements covering several categories.

3. **`reports/strict_allowlist.json::top_foreign_names`**: should NOT contain any city names. Persons (Duterte, Albayalde) may still need a separate fix if they slip past the existing pseudonymizer, but cities/provinces/regions should be empty.

4. **`reports/merge_gpt2_into_pool.json::gpt2_rejected_by_reason`**: should now include a `response_bank_coverage` category alongside the existing `too_many_distinct_entities` and `no_candidate_reference` categories. This is the new indicator-coverage filter doing its work.

5. **`reports/faith.json::pass_rate`**: should be ≥98% (was 87.18% in v2.8.7). If still <95%, the response bank may need expansion for indicators that aren't yet covered.

## What this release does NOT do

Deliberately scope-limited to maximize delivery confidence:

- **No Unity build.** That's Thesis 3.
- **No 50-respondent pilot.** That's Thesis 3.
- **No detector retraining.** v2.9.0 keeps the v2.8.7 detector training protocol unchanged. The held-out eval uses already-trained models.
- **No decoder-time rule-constrained generation.** The architecture's "rule-constrained generation" claim still uses post-hoc QLattice scoring (script 13), not constrained decoding. A real fix would require modifying the GPT-2 generation loop with rejection sampling against the QLattice prediction; this is deferred as it's a multi-week change.
- **No paper-text rewrite.** The audit recommended toning down certain claims ("rule-constrained generation" → "QLattice-scored generation"). The paper text is the team's responsibility; this release only makes the *artifact* match the more-conservative claim.
