## Why this release exists

v2.9.0 was clean on Colab T4 (where torch is preinstalled) but had two rough edges for local development on Windows + Python 3.13:

1. **`tests/test_degnn_graph.py` errored out** at collection time because it had a hard `import torch` statement. Locally, where torch isn't installed (it's a 2 GB download), this caused `pytest tests/ -q` to fail with `ModuleNotFoundError: No module named 'torch'` instead of cleanly skipping.

2. **`requirements.txt` didn't separate dev vs. full install paths** clearly. On Python 3.13, `pip install -r requirements.txt` would fail on `feyn` (no 3.13 wheels) and several other heavy deps, even when the user just wanted to run the lightweight test suite.

Both are environment issues, not architecture issues. v2.9.1 fixes them without touching the v2.9.0 audit-response code.

---

## What changed (v2.9.0 → v2.9.1)

### FIXED: `tests/test_degnn_graph.py`
Replaced `import torch` (line 20) with:
```python
torch = pytest.importorskip(
    "torch",
    reason="torch not installed — DE-GNN tests are Colab-only.",
)
```

When torch is missing, the entire file is cleanly skipped with `1 skipped` instead of erroring at collection. On Colab (where torch ships preinstalled), all tests run normally — no behavior change there.

### NEW: `dev-requirements.txt`
A lightweight install path for running the test suite locally without downloading torch, transformers, accelerate, sentencepiece, or feyn.

```bash
pip install -r dev-requirements.txt
python -m pytest tests/ -q
# → 231 passed, 1 skipped
```

Includes only: pytest, pydantic, numpy, pandas, fsspec, datasets, joblib. Total install: ~150 MB instead of ~3 GB.

### UPDATED: `requirements.txt`
- Added a top-level **PYTHON COMPATIBILITY** block documenting which Python versions are supported (3.10/3.11/3.12) and which aren't (3.13 — feyn has no wheels).
- Documented the `feyn` install fallback path: if `pip install feyn` fails, scripts/08 auto-detects this and uses sklearn LogisticRegression in the same `logreg(...)` output format. Pipeline still works.
- Added inline torch local-install hint: `pip install torch --index-url https://download.pytorch.org/whl/cpu` for CPU-only local installs (~700 MB instead of ~2 GB).
- Cross-referenced `dev-requirements.txt` so users know which file to use when.
